Installer un serveur local

Dans ce tutoriel, nous allons voir comment installer un serveur local pour effectuer nos tests avec nos futurs scripts !

danger

Ce tutoriel est pour Windows seulement !

attention

Pour avoir des droits admins sur le serveur local, vous devez suivre le deuxième tutoriel

Étape 1 : Installer l'artifact​

Artifact pour Windows

Même s'il est toujours préférable de télécharger le fichier "LATEST RECOMMENDED", nous allons télécharger la dernière version disponible.

Étape 2 : Transférer l'artifact​

2.1 Créer un dossier pour le serveur local​

Il faut créer un dossier pour déposer les fichiers du serveur. Je conseille fortement de le placer à la racine de votre disque

2.2 Le dossier artifact​

Nous allons maintenant créer un dossier dans celui que nous venons juste de faire : artifact

2.3 Transférer le contenu du .zip que nous avons téléchargé​

Il faut maintenant transférer le contenu du fichier .zip dans le dossier artifact

2.4 Télécharger le template de l'Épopee​

Cliquez ici pour télécharger

2.5 Transférer le template dans data​

Transférez le fichier "data" qui est dans le .zip dans la racine du serveur

Étape 3 : Configurer la base du serveur​

Dans les fichiers que nous venons de transférer, il y en a un qui s'appelle FXServer.exe. Il faut double-cliquer dessus

Si tout va bien, vous devriez voir une fenêtre et un site sur votre navigateur qui devraient être apparus

Étape 4 : La première connexion à TxAdmin​

4.1 Cliquer sur "Link Account"​

4.2 Connectez votre compte CFX​

ATTENTION

Il est possible que vous deviez vous connecter manuellement ! Je n'ai pas eu besoin de le faire, car j'étais déjà connecté à mon compte CFX

Assurez-vous de connecter le bon compte :)

4.3 Mettre un mot de passe​

Pour cette étape, vous pouvez mettre le mot de passe que vous voulez ! Étant donné que votre serveur local ne sera jamais mis en ligne, il n'y a aucun risque de sécurité

info

Je vous conseille de mettre votre ID Discord

Étape 5 : Les paramètres de base​

5.1 Welcome​

5.2 Server Name​

5.3 Deployment type​

info

Si vous voulez commencer de zéro, vous pouvez cliquer sur "Remote URL Template" et mettre le lien ci-dessous. https://raw.githubusercontent.com/Qbox-project/txAdminRecipe/main/qbox.yaml

5.4 Existing Server Data​

Nous devons maintenant indiquer le chemin vers le dossier data de l'étape 2.5

info

Pour obtenir le chemin vers le dossier, vous pouvez aller dans celui-ci et cliquer sur la barre de navigation, puis faire CTRL + C pour copier le chemin

5.5 Server CFG File​

Si vous avez bien suivi les étapes, vous devriez voir le même message que moi

5.6 Finish​

Il suffit juste de cliquer sur le beau bouton vert !

Étape 6 : Paramètres finaux​

6.1 Changer le temps maximum pour le démarrage d'une ressource​

Chemin pour cette étape : Settings -> Restarter

Il faut changer "Resource Starting Time Limit" pour 10 minutes

danger

N'oubliez pas de cliquer sur "Save Monitor/Restarter Settings" quand c'est fait

6.2 Ajouter votre clé CFX​

  • Allez sur ce site et connectez votre compte Fivem

  • Cliquez sur "New server" et remplissez les champs comme moi

  • Une fois la clé générée, copiez-la

  • Dans le fichier data -> "server.cfg", remplacez "TA CLÉ ICI" par la clé que vous venez de générer (ligne 9)

info

Vous pouvez aller sur ce site pour connaître votre IP publique

Étape 7 : Serveur SQL Local​

7.1 Télécharger MariaDb​

https://mariadb.org/download/?t=mariadb&p=mariadb&r=11.3.2&os=windows&cpu=x86_64&pkg=msi&mirror=osuosl

danger

Il suffit de laisser les paramètres par défaut lors de l'installation. Cependant, à la question sur le mot de passe, il faut mettre Passw0rd

Bravo, vous avez réussi !​

Vous pouvez maintenant redémarrer le serveur ! Si vous avez un message d'erreur qui n'est pas mentionné dans ce tuto, n'hésitez pas à me contacter

Petits détails 1​

Petits détails 2​
